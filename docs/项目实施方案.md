# Atlas 项目实施方案

> 版本：v1.1 · 2026-06 · **决策已确认**
> 目标：在 MVP（`atlas_demo`）验证产品形态后，启动正式开发——小程序 `atlas_wechat` + 服务端 `atlas_backend`。

---

## 1. 项目背景

### 1.1 已完成（MVP）

`atlas_demo` 以 **100% Mock 数据** 跑通了完整产品链路：

| 模块 | 能力 |
|------|------|
| 策略页 | 7 市场 × 3 策略推荐、指数 K 线、周期/翻转、右滑加自选 |
| 自选页 | 列表、市场筛选、周期同步、左滑移除 |
| 历史页 | 移出自选后盈亏复盘、胜率 KPI |
| 详情页 | K 线、企业速览、画像、雷达、四维罗盘、关键指标 |
| 搜索 | 全库模糊搜索、加自选 |

**结论：交互与信息架构已验证，可冻结为 UI/UX 参照，不再在 demo 上叠功能。**

### 1.2 正式开发目标

| 项目 | 定位 |
|------|------|
| `atlas_demo` | **产品原型 / 演示版**，只修 bug，不改需求 |
| `atlas_wechat` | **正式小程序**，对接真实 API，支持登录与用户数据 |
| `atlas_backend` | **API 服务**，数据爬取、策略筛选、用户与自选持久化 |

### 1.3 首期范围（Phase 1）

**做：**

- **A 股**真实 API（策略推荐、K 线、搜索）
- 其他市场 Tab **展示但置灰**，点击提示「即将开放」
- **微信登录必选**（自选、历史）
- 详情页含 **四维罗盘**（`/stock/{code}/compass` + 聚合 detail API）
- K 线读 **MySQL 已有 cache**，不引入 Redis

**暂缓（Phase 2+）：**

- 港股/美股/crypto 等多市场真实数据
- 历史复盘服务端归档（Phase 1 可仍用本地 storage，登录后可见）
- WebSocket 实时推送
- Redis 缓存（后期优化）

## 4. 现状差距（Backend ↔ Demo）

| Demo 能力 | Backend 现状 | 差距 |
|-----------|-------------|------|
| 3 策略 × 多市场推荐 | 1 个 API `/stock/findMy`，3 种 strategy 枚举，**仅 A 股** | 策略 ID 映射、响应 DTO 扩展、按 market 过滤 |
| 指数 K 线 | 无 | 新增指数行情接口或 Phase 1 仍用 mock |
| 个股 K 线（年/月/周/日） | DB 有 `dayk/weekk/...`，**无 REST** | `GET /stock/{code}/klines` |
| 搜索 | 无 | `GET /stock/search` |
| 详情（画像/雷达/罗盘） | 仅 `mainBusiness` | Phase 2 财报爬虫 + 聚合 API；Phase 0 可返回简化 profile |
| 自选 | 纯本地 storage | 用户表 + watchlist CRUD |
| 历史复盘 | 纯本地 storage | Phase 1 可选服务端 history 表 |
| 微信登录 | 无 | code2session + JWT |

**Backend 关键阻塞（必须先修）：**

1. **缓存未加载**：`RealtimeStockCache` 仅在 JVM `-Dproject=app` 时预热 → 正常启动 `/findMy` 返回空
2. **定时任务全部注释**：爬虫与 `recommendSave()` 无 cron
3. **仅 1 个 REST Controller**：需扩展 API 面

## 5. 分阶段实施

### Phase 0：工程初始化（1 周）

**目标：** 三套工程可独立运行、联调通路打通。

| 任务 | 负责 | 产出 |
|------|------|------|
| 从 `atlas_demo` 复制初始化 `atlas_wechat` | 前端 | 新目录，`projectname=atlas_wechat` |
| 新增 `utils/config.js`（baseUrl 分环境） | 前端 | dev 指向 `http://127.0.0.1:9010/tts` |
| 新增 `utils/api.js` + 请求封装 | 前端 | 统一 `{ code, data, message }` 适配 |
| 修复 backend 缓存启动 | 后端 | 去掉 `project=app` 门控或写入 `application.properties` |
| 本地 MySQL + 跑通 `mvn compile` + 启动 | 后端 | README 补充 `-Dproject=app` 或等价方案 |
| 微信开发者工具：勾选「不校验合法域名」 | 前端 | 本地联调 |
| 定义 OpenAPI 草案 | 全员 | `docs/api/openapi.yaml` |

**Phase 0 完成标准：** 小程序 `wx.request` 能调通 backend 健康检查或 `/stock/findMy` 并展示原始 JSON。

---

### Phase 1：策略 + K 线 + 搜索（2–3 周）

**目标：** 策略页、详情页 K 线、搜索走真实 API；**A 股 only**。

#### 1.1 后端 API

| 方法 | 路径 | 说明 |
|------|------|------|
| GET | `/stock/findMy` | **已有**，扩展响应字段：`tags`、`summary`、`resonance`、`changePct` |
| GET | `/stock/search?q=&limit=` | 从 `base` 表模糊查 code/name |
| GET | `/stock/{code}` | 基础行情：name, price, changePct, mainBusiness |
| GET | `/stock/{code}/klines?period=week&limit=50` | 返回 OHLC 数组，对齐 demo `klines` 结构 |
| GET | `/strategies` | 返回策略元数据，映射 demo 的 `trend/rebound/multi` |

**策略映射建议：**

| Demo ID | Backend StrategyTypeEnum | 说明 |
|---------|---------------------------|------|
| `trend` | `qsn` | 趋势 |
| `rebound` | `default` 或新建 | 深坑反弹 |
| `multi` | `cross_band_pressure` 或新建 | 多周期共振 |

#### 1.2 前端改造（atlas_wechat）

| 文件 | 改动 |
|------|------|
| `pages/index/index.js` | `loadRecommendations()` 调 API；保留 mock fallback 开关 `config.useMock` |
| `utils/adapter.js` | Backend DTO → demo 卡片字段 |
| `pages/detail/detail.js` | K 线 API 拉取；profile 暂用 `mainBusiness` + 静态模板 |
| `utils/search.js` | 改为远程搜索 |
| `components/*` | **不改**，保持 prop 契约 |

#### 1.3 数据管道

- 启用 `DayAllSpiderScheduler`（或等价 cron）每日收盘后跑批
- `StockTargetScheduler.recommendSave()` 定时写入 `strategy_stock`
- 文档记录：雪球 Cookie 轮换流程

**Phase 1 完成标准：** 策略页展示 backend 真实 A 股推荐；点进详情能看到真实 K 线；搜索能搜到 `base` 表里的股票。

---

### Phase 2：用户体系 + 自选同步（2 周）

**目标：** 微信登录，自选云端持久化。

#### 2.1 后端

| 方法 | 路径 | 说明 |
|------|------|------|
| POST | `/auth/wx/login` | body: `{ code }` → `{ token, openid }` |
| GET | `/watchlist` | 当前用户自选列表 |
| POST | `/watchlist` | 添加 `{ code, strategy?, entryPrice? }` |
| DELETE | `/watchlist/{code}` | 移除 |
| GET | `/user/me` | 用户信息（可选） |

**表设计（新增）：**

```sql
-- user
openid PK, nickname, avatar, created_at

-- watchlist
id, openid, code, strategy, entry_price, added_at, UNIQUE(openid, code)
```

#### 2.2 前端

| 改动 | 说明 |
|------|------|
| `app.js` | 启动时 `wx.login` → 换 token → 存 storage |
| `utils/api.js` | 自动带 `Authorization` |
| `utils/watchlist.js` | 读写改 API；本地 cache 做 offline 降级（可选） |
| 未登录 | 自选操作提示登录 |

**Phase 2 完成标准：** 换设备登录后自选列表一致；加/删自选实时同步服务端。

---

### Phase 3：详情深化 + 历史（2–3 周）

**目标：** 详情页企业认知模块逐步真实化；历史复盘可选上云。

#### 3.1 后端

| 方法 | 路径 | 说明 |
|------|------|------|
| GET | `/stock/{code}/detail` | 聚合：profile、stage、health、radar、keyMetrics |
| GET | `/stock/{code}/compass` | 四维罗盘年度序列（可先基于 `StockQuarterReport`） |
| POST | `/history/archive` | 移出自选时归档（替代纯本地） |
| GET | `/history` | 历史列表 + summary |

**数据：** 启动 `StockQuarterReport` 爬虫或对接第三方财报 API。

#### 3.2 前端

- `detail-mock.js` → `detail-api.js`，保留 mock adapter 便于开发
- 历史页 `onShow` 拉服务端 summary

**Phase 3 完成标准：** 详情页至少 3 个模块来自真实数据；历史 KPI 与服务端一致。

### Phase 4：生产化（2 周）

| 项 | 内容 |
|----|------|

## 7. API 响应规范（统一契约）

Backend 现有格式：

```json
{ "statusCode": "200", "errorMessage": "success", "data": { ... } }
```

**建议：** 新增 `ApiResponseAdvice` 或在 gateway 层统一为：

```json
{ "code": 0, "message": "ok", "data": { ... } }
```

小程序 `api.js` 做一层 normalize，兼容过渡期两种格式。

## 10. 产品决策（已确认 ✅）

| # | 问题 | 决策 |
|---|------|------|
| 1 | Phase 1 非 A 股市场 Tab | **保留展示，置灰不可切换**；点击 Toast「该市场即将开放」 |
| 2 | 自选是否必须登录 | **必须登录**；未登录自选/历史页展示登录引导 |
| 3 | 详情四维罗盘 | **Phase 1 必做**；backend 需提供 compass 年度序列 API |
| 4 | Redis K 线缓存 | **MVP 不做**；沿用 MySQL + `RealtimeStockCache`，后期优化 |

**实现位置（Phase 0 已落地）：**

- `atlas_wechat/utils/config.js` — `enabledMarkets`、`requireLoginForWatchlist`
- `atlas_wechat/components/market-tabs` — `disabled` 样式与 `disabled` 事件
- `atlas_wechat/utils/auth.js` — 登录与 `promptLogin`
- `atlas_wechat/app.js` — 自选前校验登录
